# 数据仓库设计

数据仓库是一个面向主题的、集成的、相对稳定的、反映历史变化的数据集合，用于支持管理决策。

## 核心概念

### 数据仓库特点

- **面向主题**: 按照业务主题组织数据
- **集成性**: 来自不同数据源的统一整合
- **非易失性**: 数据一旦进入仓库很少修改
- **随时间变化**: 记录历史数据的变化

### 分层架构

```
ODS (操作数据层)
    ↓
DWD (明细数据层)
    ↓
DWS (服务数据层)
    ↓
ADS (应用数据层)
```

## 数据建模

### 维度建模

#### 星型模型

```
        事实表
       /   |   \
   维度表 维度表 维度表
```

#### 雪花模型

```
        事实表
       /   |   \
   维度表 维度表 维度表
       |         |
   维度表     维度表
```

### 事实表设计

```sql
-- 订单事实表
CREATE TABLE fact_order (
    order_id BIGINT,
    user_id BIGINT,
    product_id BIGINT,
    category_id BIGINT,
    order_date DATE,
    amount DECIMAL(10,2),
    quantity INT,
    create_time TIMESTAMP,
    update_time TIMESTAMP
) PARTITIONED BY (dt STRING)  -- 按日期分区;

-- 数据加载
INSERT INTO fact_order PARTITION(dt='20240120')
SELECT
    order_id,
    user_id,
    product_id,
    category_id,
    order_date,
    amount,
    quantity,
    create_time,
    update_time
FROM ods_orders
WHERE TO_CHAR(create_time, 'yyyyMMdd') = '20240120';
```

### 维度表设计

```sql
-- 用户维度表
CREATE TABLE dim_user (
    user_id BIGINT,
    user_name STRING,
    gender STRING,
    age INT,
    city_id INT,
    city_name STRING,
    province STRING,
    register_date DATE,
    is_active BOOLEAN,
    start_date DATE,
    end_date DATE
);

-- 缓慢变化维度
INSERT INTO dim_user
SELECT
    user_id,
    user_name,
    gender,
    age,
    city_id,
    city_name,
    province,
    register_date,
    is_active,
    DATE('2024-01-20') AS start_date,
    DATE('9999-12-31') AS end_date
FROM ods_users
WHERE update_time >= '2024-01-20';
```

## ETL 流程

### 数据抽取

```sql
-- 全量抽取
INSERT OVERWRITE TABLE ods_user_full
SELECT * FROM source_users;

-- 增量抽取
INSERT INTO ods_user_increment
SELECT * FROM source_users
WHERE update_time > '${last_run_time}';
```

### 数据转换

```sql
-- 数据清洗
INSERT INTO dwd_user_clean
SELECT
    user_id,
    TRIM(user_name) AS user_name,
    UPPER(gender) AS gender,
    COALESCE(age, 0) AS age,
    city_id,
    city_name,
    province,
    register_date,
    is_active
FROM ods_user_increment
WHERE user_id IS NOT NULL
  AND user_name IS NOT NULL;
```

### 数据加载

```sql
-- 加载到汇总表
INSERT INTO dws_user_daily_summary
SELECT
    user_id,
    dt,
    COUNT(*) AS order_count,
    SUM(amount) AS total_amount,
    AVG(amount) AS avg_amount
FROM dwd_order
WHERE dt = '${current_date}'
GROUP BY user_id, dt;
```

## 性能优化

### 分区策略

```sql
-- 按日期分区
PARTITIONED BY (dt STRING)

-- 按月分区
PARTITIONED BY (year STRING, month STRING)

-- 查询时利用分区
SELECT * FROM fact_order
WHERE dt = '20240120';  -- 只扫描一个分区
```

### 分桶策略

```sql
-- 分桶
CLUSTERED BY (user_id) INTO 32 BUCKETS

-- 分桶+排序
CLUSTERED BY (user_id) SORTED BY (order_date) INTO 32 BUCKETS
```

### 索引优化

```sql
-- 物化视图
CREATE MATERIALIZED VIEW mv_user_summary
REFRESH FAST ON COMMIT
AS
SELECT
    user_id,
    SUM(order_amount) AS total_amount,
    COUNT(*) AS order_count
FROM fact_order
GROUP BY user_id;

-- Bitmap 索引
CREATE INDEX idx_user_gender ON dim_user(gender);
```

## 数仓主题设计

### 常见主题域

```
1. 用户域
   - 用户画像
   - 用户行为
   - 用户生命周期

2. 商品域
   - 商品信息
   - 商品分类
   - 商品库存

3. 交易域
   - 订单
   - 支付
   - 退款

4. 营销域
   - 活动效果
   - 优惠券
   - 推广
```

### 指标体系

```sql
-- 原子指标
CREATE TABLE dwd_order_amount (
    order_id BIGINT,
    user_id BIGINT,
    amount DECIMAL(10,2),
    dt STRING
);

-- 派生指标
CREATE TABLE ads_order_kpi (
    dt STRING,
    total_amount DECIMAL(18,2),      -- 总销售额
    order_count BIGINT,              -- 订单数
    avg_order_amount DECIMAL(10,2),  -- 均单值
    conversion_rate DECIMAL(5,4)     -- 转化率
);

-- 复合指标
CREATE TABLE ads_business_overview (
    dt STRING,
    gmv DECIMAL(18,2),                  -- GMV
    active_users BIGINT,                -- 活跃用户数
    retention_rate DECIMAL(5,4),        -- 留存率
    ltv DECIMAL(10,2)                   -- 生命周期价值
);
```
